# Packaging Count — ระบบบันทึกการตรวจนับ Packing (Fresh Food / Transfer / Non Fresh)

เวอร์ชันนี้รวมทุกอย่าง (HTML + CSS + JavaScript + ข้อมูลสินค้า/สาขา + การตั้งค่า Firebase + โลโก้ Makro)
ไว้ใน **ไฟล์เดียว** คือ `index.html` — ใส่ค่า Firebase ของคุณ (`packing-cost`) ไว้ให้เรียบร้อยแล้ว
**พร้อมใช้งานได้ทันที ไม่ต้องแก้โค้ดเพิ่ม**

## อัปเดตล่าสุด

- เปลี่ยนชื่อระบบเป็น **Packaging Count** ทั้งหมด
- เปลี่ยนไอคอน "PC" เดิม เป็นโลโก้ **Makro** (หน้า Login และ Sidebar)
- หน้า Login ซ่อนข้อความ Username/Password เริ่มต้น (ไม่แสดงให้สาธารณะเห็นแล้ว)
- เพิ่มเมนู **"แดชบอร์ด"** (หน้าแรกของทั้งสาขาและ Admin):
  - กราฟวงกลม (Donut) สรุปมูลค่าแยกตามหมวดหมู่ FRESH FOOD / TRANSFER / NON FRESH พร้อม % และมูลค่า (บาท)
  - การ์ดสรุป 4 ใบ: FRESH FOOD, TRANSFER, NON FRESH, TOTAL
  - Filter ดูแบบ "รายเดือน" หรือ "ช่วงวันที่"
  - ฝั่ง Admin เลือก Filter ดูเฉพาะสาขาใดสาขาหนึ่ง หรือดูรวมทุกสาขาได้
- เพิ่มเมนู Admin **"ล้างข้อมูลทั้งหมด"** — ลบข้อมูลตรวจนับของทุกสาขาในเดือนที่เลือกออกทั้งหมด
  (ต้องพิมพ์ยืนยันเดือนก่อนลบ และมีการบันทึก Log ไว้เสมอ)

## วิธีติดตั้ง (ง่ายที่สุด — 2 ขั้นตอน)

### ขั้นตอนที่ 1: อัปโหลดไฟล์ขึ้น GitHub

ไปที่ repository `Packing-Count` ของคุณ (https://github.com/inventoryaudit999-collab/Packing-Count)
แล้วอัปโหลดไฟล์ `index.html` นี้ **ทับไฟล์เดิม** ที่อยู่ที่ root ของ repo
(ลบโฟลเดอร์ `css/` และ `js/` เดิมออกได้เลย เพราะไม่ได้ใช้แล้ว — ทุกอย่างอยู่ในไฟล์เดียวนี้ทั้งหมด)

รอ 1-2 นาที ให้ GitHub Pages อัปเดต แล้วเปิด:
```
https://inventoryaudit999-collab.github.io/Packing-Count/
```

### ขั้นตอนที่ 2: ตั้งค่า Database Rules (สำคัญมาก — ทำครั้งเดียว)

ไปที่ [Firebase Console](https://console.firebase.google.com) → โครงการ **packing-cost**
→ เมนูซ้าย **Build → Realtime Database → Rules** → ลบของเดิมทั้งหมด แล้ววางโค้ดนี้แทน → กด **Publish**

```json
{
  "rules": {
    "items": {
      ".read": true,
      ".write": true
    },
    "counts": {
      ".read": true,
      ".write": true
    },
    "logs": {
      ".read": true,
      ".write": true,
      ".indexOn": ["ts"]
    }
  }
}
```

> หากไม่ทำขั้นตอนนี้ ระบบจะใช้งานได้ชั่วคราวตาม "Test mode" ของ Firebase (30 วันหลังสร้างโปรเจกต์)
> แล้วจะหยุดทำงานหลังจากนั้น — แนะนำให้ตั้งค่านี้ตั้งแต่ตอนนี้เพื่อให้ใช้งานได้ต่อเนื่อง
> (กฎนี้เปิดให้อ่าน/เขียนได้โดยไม่ต้อง login Firebase — ระบบจำกัดสิทธิ์ด้วย Username/Password
> ของแอปเองแทน เหมาะสำหรับใช้งานภายในองค์กร)

## เกี่ยวกับปัญหาที่พบ (หน้า Login ไม่มีสไตล์ / เข้าไม่ได้)

จากภาพที่ส่งมา หน้าเว็บแสดงทั้งหน้า Login และหน้าแอปหลักซ้อนกันแบบไม่มีสไตล์เลย — สาเหตุคือ
ไฟล์ `css/style.css` และ `js/*.js` (ที่อยู่คนละไฟล์คนละโฟลเดอร์) **โหลดไม่ขึ้น** เมื่ออัปโหลดขึ้น
GitHub (มักเกิดจากโฟลเดอร์ `css/`, `js/` ไม่ถูกอัปโหลดไปด้วย หรือ path เพี้ยน)

เวอร์ชันไฟล์เดียวนี้แก้ปัญหานี้แบบเบ็ดเสร็จ — ไม่มีไฟล์ย่อยให้หลงเหลือหรือหาย อัปโหลดไฟล์เดียว
ก็ใช้งานได้ครบทุกฟีเจอร์ทันที

## สรุปฟีเจอร์

- **หน้าสาขา**: บันทึกตรวจนับรายวัน (FRESH FOOD / TRANSFER แยก F&V-BUT-FISH, NON FRESH ช่องเดียว)
  แก้ไขย้อนหลังได้, คำนวณรวม/มูลค่าอัตโนมัติ, ค้นหาประวัติรายเดือน, Export Excel
- **หน้า Admin**: ภาพรวมสถานะบันทึกของทุกสาขา, ค้นหา/แก้ไข/ลบข้อมูลทุกสาขา, จัดการรายการสินค้า
  (เพิ่ม/แก้ไข/ลบ มีผลทันทีกับทุกสาขา), Log การใช้งานทั้งหมด, Export ข้อมูลหลายสาขาพร้อมกัน
- โครงสร้างข้อมูลจัดเรียงตามวันที่ก่อน (`/counts/{date}/{store}/...`) ทำให้ admin อ่าน/Export ข้อมูล
  ของทุกสาขาในวันเดียวได้ด้วย query เดียว รองรับ 200+ สาขาได้รวดเร็ว

ไฟล์ `database.rules.json` ที่แนบมาด้วย คือโค้ดเดียวกันกับด้านบน เผื่อสะดวกในการคัดลอก
